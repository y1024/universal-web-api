# 更新日志

## Unreleased

### 修复

- 修复全局网络监听停止超时后仍被视为成功的问题，避免残留停止线程在标签页交给新任务后关闭新监听器；监听无法安全停止时会终止交接并将标签页移出可调度状态。
- 修复独立 Cookie 标签页重绑后异步重启监听与任务获取之间的竞态，监听注册、标签页状态检查和任务交接现在通过专用监听转换锁串行化。
- 修复空闲标签页异步回收期间，尚未关闭的浏览器 target 被扫描器重新加入池、随后又被旧关闭任务误关的问题；待关闭 target 会保留墓碑并自动重试，直到浏览器快照确认其消失。
- 修复 `min_tabs` 将忙碌标签页计入最小可用容量的问题，空闲回收现在会实际保留配置数量的空闲标签页。
- 为 Responses API 实现进程内有界、带过期时间的响应状态存储，`previous_response_id` 现在可以恢复上一轮消息和函数调用上下文，`function_call_output` 不再降级为普通用户文本；`store=false` 不会留下可复用状态。
- 修复函数调用校验重试耗尽后仍可能向客户端返回未知工具、非法参数或违反 `tool_choice` 的调用；失败降级默认关闭，即使显式启用也只允许已经通过校验的调用。
- 使用标准 JSON Schema 校验器支持本地 `$ref` / `$defs` 等常见 schema，并修复超大整数校验时转换为浮点导致 `OverflowError` 的问题；外部 schema 引用会被明确拒绝。
- 移除 Anthropic 兼容层跨请求共享的工具名称缓存，避免客户端复用 `tool_use_id` 时继承其他请求的工具名。
- 修复旧版 `functions` 请求显式开启并行调用时，响应格式只能表示第一个调用而静默丢失其余调用的问题；旧版协议现在始终强制单调用。
- 修复 DeepSeek 开启网页搜索时搜索/工具片段的 `FINISHED`、`finish`、`close` 信号被误判为整条回复结束，导致网络流提前中断并返回空内容的问题；该修复关联已完成的 Issue #20。
- 为 MiMo、Qwen、豆包等解析器补充“已见到可见正文后才接受 done-only 结束信号”的保护，降低函数调用、思考或搜索等中间态完成事件提前截断后续回答的风险。
- 网络监听器现在会在支持 DOM 回退的解析器返回“空内容完成”时转入 DOM 监听补齐，同时保留已提取到图片/媒体结果的正常完成路径。

### 改进

- 合入来自 PR #21 的附件上传状态误判修复：附件监控现在支持 `ignored_busy_text_markers`，可把输入区固定文案或开关 token 从忙碌文本匹配中排除，避免 Qwen 等站点的 `thinking` 被误判为附件仍在处理。
- 在文件粘贴配置面板中新增“忽略忙碌文本 / token”配置项，并同步更新后端 schema、配置校验和站点配置。
- 参考 PR #18 中的性能优化思路，为配置引擎新增轻量 TTL 缓存，缓存高频读取的默认预设、站点配置和流式配置；配置保存或热重载后会自动失效，避免读取旧数据。
- 优化调试日志节流锁粒度，并整理少量热路径导入，减少不同节流键之间的锁竞争。

### 测试

- 新增标签页监听停止超时、异步重绑交接、待关闭 target 防重入和空闲容量下限的确定性并发回归测试。
- 新增 Responses 函数调用续接、`store=false`、Anthropic 请求隔离、工具降级、JSON Schema 引用/超大整数及旧版并行调用的回归测试。
- 新增附件监控忽略忙碌文本的回归测试。
- 新增配置缓存行为测试，覆盖 `None` 值缓存和 TTL 过期失效。
